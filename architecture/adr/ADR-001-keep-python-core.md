# ADR-001：保留 Python Core

| 项目 | 内容 |
| --- | --- |
| 状态 | Accepted |
| 日期 | 2026-04-24 |
| 决策 | 系统核心继续使用 Python，不进行 Node.js 全量迁移 |

## 背景

当前项目已经基于 Python、FastAPI 和 Playwright 建立了可运行的采集能力。后续系统将进一步扩展为稳定采集框架、数据处理中心和订阅推送中心，因此需要确认核心技术路线。

## 决策

系统核心继续使用 Python，保留现有 Python 采集内核，并在此基础上继续扩展调度治理、数据处理和订阅推送能力。

## 理由

| 理由 | 说明 |
| --- | --- |
| 现有基础成熟 | 当前仓库已经基于 Python 落地了主要能力 |
| 迁移成本低 | 无需重写采集、登录态、调度和策略层 |
| 更适合数据处理 | 清洗、去重、分类、评分更适合继续在 Python 内核中演进 |
| Playwright 支持足够 | Python 版本能力完整，满足当前抓取需求 |
| 风险更低 | 当前复杂度在采集治理，不在前端或桌面栈 |

## 替代方案

| 方案 | 问题 |
| --- | --- |
| 全量迁移到 Node.js / Electron | 重写成本高，短期不会带来稳定性收益 |
| 同时维护双内核 | 成本高，边界复杂，不适合当前阶段 |

## 结果

| 结果 | 说明 |
| --- | --- |
| 正面结果 | 可以把主要精力集中在稳定采集和数据处理能力 |
| 约束 | 后续如果增加桌面壳，应以 Python Core 为中心对接 |
