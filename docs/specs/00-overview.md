# 00 系统总览与基座定位

状态：已落地（v1）

## 00.1 项目定位

| 项目     | 内容                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 名称     | 热点信息采集系统                                                           |
| 当前阶段 | 由"内部工具"演进为"运维同学托管的可持续基座"                               |
| 目标用户 | 运营（每日热点收集）、运维（部署/配置/排障/升级）、开发（扩展采集源/策略） |
| 核心价值 | 自动化每日热点采集、过滤、汇总、分发；运维侧零脚本即可启停、备份、升级     |

## 00.2 角色与关注点

| 角色 | 主路径                                 | 核心关注                                    |
| ---- | -------------------------------------- | ------------------------------------------- |
| 运营 | 首页 / 采集源 / 报告                   | 任务是否成功、报告是否准时、Cookie 是否过期 |
| 运维 | `/system/*` 页面 + `scripts/`          | 启停、备份、版本、健康指标、配置脱敏导出    |
| 开发 | `app/services/strategies/*` + `tests/` | 新增策略、修复解析、回归测试                |

## 00.3 不在范围

- 多租户与权限模型（默认信任内网部署）
- AI 摘要 / 自动打分（保留接口扩展点，不实现）
- Docker 与分布式 worker（保留单机 + onedir 形态）
- 企业 SSO（保留本地配置文件 + 环境变量）

## 00.5 关键非功能要求

| 维度   | 要求                                                          |
| ------ | ------------------------------------------------------------- |
| 启停   | 双击脚本即可启停；`scripts/stop.ps1` 优雅释放 PID 与端口      |
| 备份   | 一键备份/恢复 SQLite，自动保留近 14 份                        |
| 升级   | 升级包覆盖后启动自动迁移数据库，不丢运行数据                  |
| 可观测 | `/system/info` + `/system/health/extended` 提供版本与健康指标 |
| 安全   | 敏感配置默认掩码导出；可选 Fernet 加密；URL 仅 http/https     |
| 可维护 | 新增采集策略仅新建一个文件 + `@register("name")`，不改分发层  |
