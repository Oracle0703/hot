# B站 Cookie 网页录入与自动写入设计

## 背景

当前发布版仍存在通过编辑 `data/app.env` 手工维护 `BILIBILI_COOKIE` 的流程，实际使用中容易出现以下问题：

| 问题 | 说明 |
| --- | --- |
| 操作门槛高 | 运营同学需要理解 `app.env` 与运行目录 |
| 容易粘贴错误 | 常见错误包括误贴 `BILIBILI_COOKIE=...`、带引号、带换行 |
| 失败反馈不明确 | 保存失败时很难判断到底是格式错、缺字段还是写入目录错 |
| 发布版与源码版体验不一致 | 源码中已有 B站登录态面板，但发布版需要确保该入口实际可用 |

本次目标是把 B站 Cookie 的维护入口统一收敛到系统网页中，让用户在 B站完成操作后，把整串 Cookie 粘贴到系统自己的网页，由系统自动解析并写入 `app.env`。

## 目标

| 项目 | 目标 |
| --- | --- |
| 主入口 | 使用 `/scheduler` 页面中的 `B站登录态` 文本框 |
| 输入方式 | 用户粘贴一整串 Cookie 字符串 |
| 自动处理 | 系统自动规范化输入并写入 `BILIBILI_COOKIE` |
| 严格校验 | 如果缺少 `SESSDATA=`，直接拒绝保存 |
| 生效方式 | 保存后立即更新运行时环境，后续采集直接使用最新值 |

## 非目标

| 项目 | 说明 |
| --- | --- |
| 自动抓浏览器 Cookie | 不从浏览器自动同步，不读取本地浏览器存储 |
| 新增独立配置页 | 不新增专门的 B站 Cookie 页面，仍在 `/scheduler` 完成 |
| 修改其它配置表单 | 不顺手改钉钉、代理、调度时间等表单逻辑 |
| Cookie 加密存储 | 仍写入现有 `app.env`，不引入新的密钥管理方案 |

## 用户流程

| 步骤 | 用户动作 | 系统行为 |
| --- | --- | --- |
| 1 | 用户在 B站完成登录或其它需要的页面操作 | 无 |
| 2 | 用户从 B站复制一整串 Cookie | 无 |
| 3 | 用户打开系统 `/scheduler` 页面 | 页面展示 `B站登录态` 文本框 |
| 4 | 用户把整串 Cookie 粘贴到文本框并点击保存 | 服务端解析与校验输入 |
| 5 | 如果输入合法 | 写入 `app.env`，同步更新运行时环境 |
| 6 | 如果输入非法 | 不写入，返回当前页面并展示错误 |

## 页面设计

### 入口位置

继续使用调度页中的 `B站登录态` 面板，不增加新的导航入口。

### 表单结构

| 字段 | 设计 |
| --- | --- |
| 输入控件 | 单个 `textarea`，字段名保持 `bilibili_cookie` |
| 提交按钮 | 保持一个“保存B站登录态”按钮 |
| 说明文案 | 明确支持直接粘贴整串 Cookie，支持自动识别误贴的 `BILIBILI_COOKIE=` 前缀 |

### 说明文案要求

文案需要明确以下规则：

| 规则 | 说明 |
| --- | --- |
| 直接粘贴整串 Cookie | 不需要用户自己拼成 `BILIBILI_COOKIE=...` |
| 可自动兼容常见脏输入 | 包括前缀、引号、换行 |
| 缺少 `SESSDATA=` 将拒绝保存 | 这是硬校验 |
| 保存后立即生效 | 后续 B站采集直接用新值 |

## 后端解析设计

### 输入规范化流程

服务端在接收 `bilibili_cookie` 后，按以下顺序处理：

| 步骤 | 规则 | 目的 |
| --- | --- | --- |
| 1 | 读取原始文本 | 接住用户提交值 |
| 2 | 去首尾空白 | 消除复制残留空格 |
| 3 | 去除外层单引号或双引号 | 兼容 `"SESSDATA=..."` 这类误贴 |
| 4 | 把换行、回车、连续空白压成单个空格 | 兼容多行复制 |
| 5 | 如果以 `BILIBILI_COOKIE=` 开头，只提取等号后面的值 | 兼容误贴整行 env 配置 |
| 6 | 再次去首尾空白 | 保证最终值干净 |
| 7 | 检查是否包含 `SESSDATA=` | 缺失则视为非法输入 |

### 校验规则

| 校验项 | 结果 |
| --- | --- |
| 输入为空 | 拒绝保存 |
| 不包含 `SESSDATA=` | 拒绝保存 |
| 包含 `SESSDATA=` | 允许保存 |

本次采用严格模式，只要缺少 `SESSDATA=` 就不写入。

### 写入规则

| 项目 | 处理方式 |
| --- | --- |
| 落盘键名 | 固定写入 `BILIBILI_COOKIE` |
| 写入位置 | 当前实际运行目录对应的 `data/app.env` |
| 运行时更新 | 同步更新 `os.environ['BILIBILI_COOKIE']` |
| 写入内容 | 使用规范化后的整串 Cookie |

## 交互反馈设计

### 成功路径

| 行为 | 设计 |
| --- | --- |
| 响应方式 | 保存成功后 `303` 跳回 `/scheduler` |
| 提示方式 | 使用查询参数携带成功状态，例如 `bilibili_saved=1` |
| 页面提示 | 在 `B站登录态` 面板附近展示成功提示 |
| 提示内容 | 明确说明“B站登录态已更新”，必要时补充“已自动按标准格式保存” |

### 失败路径

| 行为 | 设计 |
| --- | --- |
| 响应方式 | 失败时不重定向，直接重新渲染调度页 |
| HTTP 状态 | 可返回 `422` |
| 页面提示 | 在 `B站登录态` 面板附近展示错误信息 |
| 输入保留 | 保留用户本次提交内容，避免长字符串丢失 |
| 安全要求 | 错误消息不回显完整 Cookie 到 URL 中 |

### 失败提示文案

| 场景 | 提示文案 |
| --- | --- |
| 输入为空 | `B站 Cookie 不能为空` |
| 缺少 `SESSDATA=` | `Cookie 缺少 SESSDATA，系统未保存` |

## 数据流

| 阶段 | 数据 |
| --- | --- |
| 页面提交 | `bilibili_cookie` 原始文本 |
| 服务端规范化 | 生成标准化 Cookie 字符串 |
| 服务端校验 | 判断是否允许保存 |
| 环境写入 | 更新 `app.env` 中的 `BILIBILI_COOKIE` |
| 运行时更新 | 更新进程内环境变量 |
| 后续采集 | B站采集策略读取最新 `BILIBILI_COOKIE` |

## 错误处理

| 错误类型 | 处理 |
| --- | --- |
| 用户输入为空 | 阻止写入，返回页面错误提示 |
| 用户输入缺少 `SESSDATA=` | 阻止写入，返回页面错误提示 |
| 环境文件不存在 | 复用现有 `AppEnvService` 的自动创建能力 |
| 写入后用户仍采集失败 | 继续沿用现有任务诊断摘要，提示登录失效或风控 |

## 测试设计

### 单元测试

| 用例 | 预期 |
| --- | --- |
| 纯 Cookie 串保存 | 原样或规范化后写入 `BILIBILI_COOKIE` |
| 带 `BILIBILI_COOKIE=` 前缀 | 自动提取值并保存 |
| 带外层引号 | 去掉引号后保存 |
| 带换行与多余空格 | 规范化为空格分隔后保存 |
| 缺少 `SESSDATA=` | 抛出明确错误，不写入 |
| 空字符串 | 抛出明确错误，不写入 |

### 集成测试

| 用例 | 预期 |
| --- | --- |
| `/scheduler` 展示 B站 Cookie 面板 | 页面包含 `bilibili_cookie` 字段 |
| 提交合法 Cookie | 重定向回 `/scheduler`，`app.env` 写入正确 |
| 提交带前缀的 Cookie | 重定向回 `/scheduler`，`app.env` 写入规范化值 |
| 提交缺少 `SESSDATA=` 的 Cookie | 不重定向，页面出现错误提示，`app.env` 不写入脏值 |

## 实现边界

| 边界 | 说明 |
| --- | --- |
| 只做网页粘贴入口 | 不做浏览器自动提取 |
| 只处理 B站 Cookie | 不引入其它站点的通用 Cookie 管理 |
| 不改采集策略协议 | 采集侧继续读取 `BILIBILI_COOKIE` |
| 不新增数据库字段 | 配置仍落在 `app.env` |

## 风险与权衡

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 用户粘贴的并不是完整 Cookie | 保存成功但采集仍可能失败 | 严格要求至少有 `SESSDATA=`，并保留现有任务诊断提示 |
| Cookie 过期很快 | 后续采集失败 | 允许用户直接在网页重新粘贴并立即生效 |
| 发布版与源码版入口不一致 | 用户继续困惑 | 本次实现需要确保发布版实际包含该网页入口 |

## 验收标准

| 编号 | 标准 |
| --- | --- |
| 1 | 用户可以在 `/scheduler` 页面直接粘贴整串 B站 Cookie |
| 2 | 系统能自动兼容前缀、引号、换行等常见误贴格式 |
| 3 | 输入缺少 `SESSDATA=` 时，系统拒绝保存且页面给出明确错误 |
| 4 | 输入合法时，系统把规范化结果写入实际运行目录的 `app.env` |
| 5 | 保存后无需重启，后续 B站采集可直接使用最新 Cookie |
