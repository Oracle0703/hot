# 首页入口按钮高度统一设计

## 背景

首页 Hero 区当前有 3 个主入口按钮：

- 立即采集
- 采集源管理
- 历史报告

这 3 个按钮虽然共用同一套按钮风格，但在实际渲染中高度表现不一致，导致首页首屏操作区视觉不整齐。

本次需求只处理这 3 个首页入口按钮的高度一致性，不扩展为全局按钮系统改造。

## 目标

- 让首页 Hero 区的 3 个入口按钮高度统一
- 保持现有按钮文案、宽度、颜色、圆角、字号不变
- 不影响其它页面按钮样式
- 优先通过局部样式规则修复，而不是重做按钮组件结构

## 方案选择

本次采用“方案 A：首页按钮区域局部统一高度”。

| 方案 | 做法 | 结论 |
| --- | --- | --- |
| A | 只对首页 Hero 区按钮增加统一高度与垂直居中规则 | 采用 |
| B | 修改全局 `.button` 高度 | 不采用，影响面过大 |
| C | 为每个按钮单独加 class | 不采用，维护成本高 |

## 设计说明

### 调整范围

仅处理首页 `/` Hero 区中的这组按钮容器：

- `立即采集`
- `采集源管理`
- `历史报告`

### 保持不变

以下内容保持不变：

- 按钮文字
- 按钮宽度策略
- 按钮主次样式区分
- 圆角
- 颜色与阴影
- 首页整体布局

### 调整内容

通过首页按钮容器下的局部样式规则，统一按钮：

- 最小高度
- 垂直居中方式
- 行高/内边距导致的视觉偏差

建议做法：

- 保持现有 `.button` 基础样式不变
- 在首页 Hero 区按钮容器下，对链接按钮和表单按钮统一使用同一高度表现
- 使用 `inline-flex` 或等效方式保证文字垂直居中

## 实现位置

| 文件 | 责任 |
| --- | --- |
| `app/ui/page_theme.py` | 补首页按钮区域局部高度统一样式 |
| `tests/integration/test_pages.py` | 增加首页按钮一致性的回归断言 |

## 不在范围内

以下内容不在本次处理范围内：

- 其它页面按钮高度
- 全局按钮系统重构
- 首页按钮排版重做
- 宽度统一或按钮等宽处理

## 验证策略

| 项目 | 验证点 |
| --- | --- |
| 首页结构 | 3 个按钮仍存在且链接/提交行为不变 |
| 样式回归 | 首页 Hero 区按钮具备统一高度规则 |
| 影响范围 | 其它页面测试不受影响 |

## 结论

这次改动是一次局部样式收口，只解决首页 3 个主入口按钮的高度不一致问题，不扩大为全站按钮体系调整。
